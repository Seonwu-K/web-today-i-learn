# 구현 사항

## 1단계 - HTML

- `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` 시맨틱 태그를 사용한 페이지 구조 구성
- 프로필 사진(`<img>`), 닉네임, 한 줄 소개로 프로필 히어로 영역 구성
- 자기소개 문단(`<p>`) 작성 및 GitHub, 블로그 링크(`<a>`) 연결
- 이름, MBTI, 취미, 좋아하는 음식 정보를 `<table>`로 정리
- 갤러리 섹션에 9장의 이미지 배치
- 인생 작품 2가지를 목록 형태로 소개하고, 그중 하나에 `<iframe>` 영상 삽입
- 날짜, 학습 주제, 내용을 포함한 TIL 섹션 구성
- `<meta charset="UTF-8">`, `<meta name="viewport">`, Open Graph 태그(`og:title`, `og:description`, `og:image`) 설정
- 모든 이미지에 `alt` 속성 작성
- 이미지, CSS, JS 파일을 상대 경로로 연결

## 2단계 - CSS

- `reset.css`, `common.css`, `wiki-layout.css` 파일 분리
- `main.css`에서 `@import`를 사용한 CSS 통합 관리
- 태그 선택자(`body`, `main`, `footer` 등), 클래스 선택자(`.content-section`, `.gallery-grid` 등), ID 선택자(`#til`) 사용
- 갤러리 9장을 `display: grid`와 `grid-template-columns: repeat(3, 1fr)`로 3열 배치
- `:root`에 CSS 변수 정의 및 색상 테마 관리
- 상단 내비게이션 sticky, blur, hover 효과 적용
- 카드, 링크, 갤러리 이미지에 transition 기반 hover 효과 적용
- `.video-wrapper`를 활용한 16:9 비율 영상 영역 구성

## 3단계 - JavaScript

- `document.querySelector()`를 사용한 TIL 폼, 목록, 입력 요소 선택
- `submit` 이벤트 리스너를 통한 TIL 항목 동적 추가
- `createElement()`와 `textContent`를 사용한 DOM 요소 생성
- `prepend()`를 사용한 최신 TIL 항목 상단 배치
- 갤러리 이미지에 `mouseenter`, `mouseleave` 이벤트 연결
- hover 시 전체 미리보기 오버레이 표시 기능 구현
- 필요한 시점에 미리보기 오버레이를 생성하는 방식 적용

# 생각해보기

## HTML

### 나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까?

`<img>` 태그가 적절하다고 생각했습니다. 현재 갤러리 이미지는 단순 장식이 아니라 사용자가 직접 감상하는 콘텐츠이기 때문에 이미지 의미를 전달할 수 있는 `alt` 속성이 필요합니다. 

### 뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?

이미지를 불러오는 방식은 `<img src="...">`로 직접 삽입하는 방식과 CSS의 `background-image`로 배경처럼 적용하는 방식으로 나뉘는 것으로 알고있습니다. 이 미션에서는 HTML 문서만 보고도 이미지 태그임을 알기 쉽도록 img 태그를 직접 사용했습니다.

### 시맨틱 태그로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?

문서의 모양보다는 각 영역의 역할과 의미를 기준으로 나누는 것이 중요하다고 생각합니다. 상단 이동 링크는 `<nav>`, 메인 콘텐츠는 `<main>`, 주제가 나뉘는 자기소개, 갤러리 부분은 `<section>`, 작품 카드나 TIL 항목처럼 독립된 내용은 `<article>`로 구분하는 방식이 적절한 것 같습니다.

### `<img>` 태그의 alt 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?

`alt` 속성은 오류로 인해 이미지를 로드하지 못했을 때, 어떤 이미지 정보가 해당 위치에 존재하는지 나타내기 위해 필요한 속성이라고 생각합니다. 따라서 아무 텍스트나 작성하기 보단, 해당 이미지가 어떤 이미지인지 나타낼 수 있는 내용으로 채워야 할 것 같습니다.

### 이미지를 로드할 때, 상대 경로와 절대 경로를 혼용하면 어떤 문제가 발생할 수 있을까?

GitHub Pages처럼 저장소 이름이 경로에 포함되는 환경에서는 절대 경로와 상대 경로를 혼용할 경우 일부 리소스가 정상적으로 보이지 않을 수 있습니다. 로컬에서는 잘 보이더라도 배포 환경에서는 이미지가 깨질 수 있기 때문에, 상대경로를 적용하는 편이 안전할 것 같습니다.

### `<meta>` 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?

viewport 설정은 모바일 환경에서 화면 너비에 맞춰 페이지가 올바르게 렌더링되도록 도와주는 역할을 한다고 이해했습니다. Open Graph 태그는 링크를 공유했을 때 제목, 설명, 이미지 같은 미리보기 정보를 제공하는 역할을 합니다.

## CSS

### 갤러리 9장 배치에 Flexbox와 CSS Grid 중 어떤 것이 더 적합할까?

이 프로젝트처럼 9장의 이미지를 3열로 정렬하는 경우에는 CSS Grid가 더 적합한 것 같습니다. FLex는 반응형으로 레이아웃을 처리하기 쉬운 방식이긴 하지만, Grid는 행과 열을 함께 제어하는 2차원 레이아웃에 용이하고 의도를 표현하기에 더 좋아 적합해보입니다.

### 태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?

- 태그 선택자는 `body`, `main`, `footer`처럼 전역 기본 스타일에 적합합니다.
- 클래스 선택자는 여러 곳에서 재사용되는 UI 단위에 적합합니다. 
- ID 선택자는 문서 내에서 하나만 존재하는 특정 요소를 지정할 때 적합합니다.

### CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?

HTML 안에 직접 작성하면 처음에는 빠르게 확인할 수 있고 간편하지만, 코드가 커질수록 구조와 스타일이 섞여 가독성이 떨어지는 문제가 있습니다. 

CSS를 별도 파일로 분리해두면 유지보수가 쉬워지고, 여러 파일에서 재사용하기도 편해진다고 생각합니다.

## JavaScript

### `document.getElementById()`와 `document.querySelector()`의 차이는 무엇인가?

`getElementById()`는 ID 하나만 빠르게 찾을 때 용이하고, `querySelector()`는 클래스, 태그, ID 등 CSS 선택자 문법 전체를 사용할 수 있습니다. 이 미션에서는 폼, 목록, 입력 요소를 일관된 방식으로 다루기 위해 `querySelector()`를 사용했습니다.

### 자기소개 페이지에서 활용할 수 있는 이벤트에는 어떤 것들이 있을까?

이번 미션 기준으로는 `submit`, `mouseenter`, `mouseleave` 같은 이벤트를 활용하고자 했습니다. `submit`은 TIL 작성 폼 등록에, `mouseenter`와 `mouseleave`는 갤러리 이미지 hover 미리보기에 사용할 수 있었습니다. 이 외에도 사용자의 동작에 따라 색다른 효과를 주려면 `click`, `scroll`, `keydown` 같은 이벤트도 활용 가능할 것 같습니다.

### `innerHTML`로 DOM을 조작할 때 주의해야 할 점은 무엇일까?

사용자 입력값을 `innerHTML`에 그대로 넣으면 의도하지 않은 HTML이 삽입될 수 있고, XSS 같은 보안 문제로 이어질 수 있습니다. 그래서 이 미션에서는 `createElement()`와 `textContent`를 사용해 요소를 생성하는 방식을 사용하였습니다.

## 배포

### GitHub Pages로 배포한 페이지에서 TIL을 작성하면, 새로고침하거나 재배포했을 때 데이터가 유지될까?

현재 코드 기준으로는 유지되지 않습니다. TIL 항목은 JavaScript로 DOM에만 추가될 뿐 정보를 저장하고 불러오는 과정이 없습니다. 새로고침을 하면 새로 추가한 항목은 사라지게 됩니다.

### JavaScript로 추가한 데이터를 영구적으로 저장하려면 어떤 방법이 있을까?

가장 간단한 방법은 `localStorage`를 사용하는 것입니다. 브라우저에 데이터를 저장해 새로고침 후에도 유지할 수 있게 됩니다.

더 확장된 방식으로는 백엔드 서버와 데이터베이스를 두고 데이터를 저장하는 방법이 있습니다.
